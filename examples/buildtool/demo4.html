Ozma.js demo4: / 示例4（支持第三方包管理系统）

for third party package manager
Oz.js兼容RequireJS模块和SeaJS模块的语法，Ozma.js也兼容第三方包管理系统，如本例中的Jam。
Source code for the demo: View on Github
How to build distribution file of the demo: (How to install Ozma.js)
cd examples/buildtool/
jam install backbone
ozma js/main_jam.js --jam -c js/ozconfig_jam.json
Running result of the demo:
  1. Open the developer console of the browser
  2. Refresh

说明：
  • 使用jam安装、更新、删除package之后，执行ozma --jam，会在./jam/（或配置文件中jamPackageDir指定的目录）下生成oz.js和oz.config.js
  • jam/oz.config.js中包含Jam package的路径配置，可单独使用，比如： 
    <script src="../../oz.js"></script>
<script src="jam/oz.config.js"></script>
<script src="js/main_jam.js"></script>
  • jam/oz.js里也包含了oz.config.js中的配置，所以可单独使用： 
    <script src="jam/oz.js"></script>
<script src="js/main_jam.js"></script>
  • 用ozma执行构建时如果包含--jam参数，会自动将jam/oz.js作为配置里的loader选项，否则会将oz.config.js的内容与开发者自定义的loader文件合并，存放在发布文件中
  • 使用Jam package时，不需要考虑module文件的真实路径，可直接用package的名字作为模块名。
